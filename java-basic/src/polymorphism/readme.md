# 다형성
- 객체지향의 대표적인 특징 : 캡슐화, 상속, 다형성
- 다형성은 객체지향 프로그래밍의 꽃
## 다형성의 핵심이론
1. 다형적 참조
   - 핵심 : 부모는 자식을 품을 수 있다.
2. 메서드 오버라이딩  
   - 오버라이딩된 메서드가 항상 우선권을 가진다.(멤버변수는 오버라이딩 x)

## 순수 추상 클래스
- 인스턴스 생성 x
- 상속한 자식은 모든 메서드를 오버라이딩 해야함.
- 주로 다형성을 위해 사용
```aiignore

public abstract class AbstractAnimal {

    public abstract void sound();

    public abstract void move();
}

```

# 인터페이스(설계도)
순수 추상 클래스와 같지만 약간의 편의 기능 추가
- 메서드는 모두 public abstract(생략 가능)
- 다중 구현 지원
- 모든 필드가 상수(static final)

# 상속 vs 구현
- 부모의 기능을 물려받는 것이 목적 -> 상속
- 모든 메서드가 구현이 필요 -> 구현

# 왜 인터페이스를 사용?
```
모든 메서드가 추상 메서드면 순수추상 클래스를 만들어도 되고, 인터페이스를 만들어도 되는데
왜 인터페이스를 사용?
```
1. 제약)
- 인터페이스의 규약은 반드시 구현해야함. 하지만 순수 추상 클래스의 경우 미래에 누군가 그곳에 실행가능한 메서드를 추가가능
2. 다중 구현
- 인터페이스는 다중 구현이 가능하다는 것

# 왜 다중 상속은 허용하지 않을까?
- 디아몬드 문제 -> 여러 클래스를 다중 상속받는 다는 가정, 같은 이름의 메서드를 여러 부모 클래스가 가질 때 자식은 무엇을 선택해야할지 알 수 없다.
