# 자바 메모리 구조
1. 메서드 영역
   - 프로그램 실행에 필요한 공통 데이터 관리
     - 클래스정보 : 클래스의 실행코드(바이트코드),필드,메서드,생성자 등 실행코드
     - static: static 변수들 보관
     - 런타임 상수 풀: 프로그램 실행에 필요한 공통 리터럴 상수 보관(ex. "hello"라는 리터럴 문자를 공통으로 묶어서 관리)
2. 스택 영역 
   - 지역변수, 중간 연산결과, 메서드 호출 정보 포함
   - 메서드 호출 시 하나의 스택 프레임, 종료되면 해당 스택 프레임 제거
3. 힙 영역
   - 객체(인스턴스)및 배열이 생성되는 영역이다.
   - GC가 이뤄지는 주요영역
```aiignore
스택 영역은 각 쓰레드 별로 하나의 실행 스택이 생성된다.
-> 쓰레드 수 만큼 스택 영역이 생성된다.
```

## 메서드 코드는 메서드 영역으로
```aiignore
특정 클래스로 100개의 인스턴스를 생성 
-> 힙 메모리에 100개의 인스턴스가 생성되지만, 메서드는 메서드 영역에서 공통으로 관리되고 실행
```
